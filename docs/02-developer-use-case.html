Project 2 — Developer Guide

DocuMind — Developer Use Case Guide

Everything you need to run, test, and extend the DocuMind RAG system locally and in production.

⚡ Quick Start — Docker (Recommended)

One-Command Startup Docker Compose starts all three services simultaneously. PyTorch and the embedding model are downloaded automatically on first build. 
# From the project root:
cd project-2-rag-documind
docker compose up --build
First Run Notes First build takes 5–10 minutes (PyTorch compilation). The all-MiniLM-L6-v2 model (~80MB) downloads automatically from HuggingFace on first startup — internet required once. Subsequent starts are fast.

Service URLs after startup

http://localhost:3001 Frontend (React app)

http://localhost:5001/swagger Backend Swagger / OpenAPI

http://localhost:8000/docs RAG Engine interactive docs

🔨 Local Development Setup (Without Docker)

Run each service individually for faster iteration during development.

1. RAG Engine (Python)

cd rag-engine
python3.11 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --port 8000 --reload

2. Backend (.NET 8)

cd backend/DocuMind.Api
dotnet restore
dotnet run                        # Starts on http://localhost:5001

3. Frontend (React + Vite)

cd frontend
npm install
npm run dev                       # Starts on http://localhost:3001
Start Order Start services in order: RAG Engine first, then Backend, then Frontend. The .NET backend will fail health checks if the RAG engine is not already running.

🎯 End-to-End Workflow

1
Upload Documents
Navigate to /upload. Drag and drop or click to upload PDF, DOCX, TXT, or MD files. Maximum 50MB per file. Multiple files can be uploaded in one batch.
2
Wait for Processing
DocuMind automatically runs the pipeline: text extraction → recursive chunking → embedding → vector storage in ChromaDB. Watch the document status update in real time.
3
Ask Questions
Navigate to /chat. Type a natural language question. DocuMind embeds the query, searches ChromaDB for semantically similar chunks, and returns an answer with source citations.
4
Review Sources
Each answer includes citation cards. Click a citation to see the exact document chunk that was used to generate the answer, including its similarity score.
5
Browse Knowledge Base
Navigate to /knowledge-base. Run semantic searches directly against your indexed documents. Filter by document, view all chunks, inspect embeddings.
6
View Analytics
Navigate to /analytics. View usage patterns, query frequency, most-accessed documents, and system performance metrics via Recharts visualizations.

🧪 Testing the RAG Pipeline Directly (curl)

Use these commands to test the RAG engine API independently from the frontend and backend.

Health Check

curl http://localhost:8000/health

Process a Document

curl -X POST http://localhost:8000/process \
  -H "Content-Type: application/json" \
  -d '{
    "file_path": "./uploads/test.txt",
    "document_id": "doc-001",
    "file_name": "test.txt"
  }'

Query the Knowledge Base

curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What is the main topic?",
    "top_k": 5,
    "score_threshold": 0.3
  }'

List Collections

curl http://localhost:8000/collections

Get Document Chunks

curl http://localhost:8000/chunks/doc-001

📋 Use Cases

UC-01
Enterprise Knowledge Base
Upload company policies, HR documents, SOPs, and internal guides. Employees can query in natural language: "What is the remote work policy?" or "How do I request time off?" — and get cited answers instantly, without reading entire documents.
UC-02
Legal Document Analysis
Upload contracts, NDAs, compliance documents. Search clauses semantically — "find all clauses about liability" — without needing to know the exact legal phrasing. Great for due diligence and legal review workflows.
UC-03
Technical Documentation Q&A
Upload API docs, runbooks, architecture guides, and incident playbooks. Engineers ask operational questions: "How do I restart the payment service?" or "What is the escalation procedure for P1 incidents?" — answered with exact source references.
UC-04
Research Assistant
Upload academic papers, research reports, market analyses. Ask high-level questions: "What are the main findings about transformer architectures?" — get source-cited summaries that point to specific sections for deep reading.
UC-05
Parameter Tuning & Experimentation
Use the /settings page to tune chunk_size, top_k, and score_threshold without restarting the service. Compare retrieval quality across different settings. Ideal for optimizing RAG performance for a specific document corpus.

🧠 LLM Integration Options

LLM is Optional DocuMind works fully without an LLM using its rule-based answer assembly. Add an LLM only if you need natural language generation (more fluent responses). The RAG retrieval quality is independent of the LLM choice.
Option A
OpenAI
Add OPENAI_API_KEY=sk-... to RAG engine .env. Recommended model: gpt-4o-mini (cheapest, ~$0.15/1M tokens). gpt-4o for highest quality. Free trial credits available.
Option B
Anthropic Claude
Add ANTHROPIC_API_KEY=sk-ant-... to RAG engine .env. Recommended model: claude-haiku-4-5 (fastest, cheapest). claude-sonnet-4-6 for highest quality.
Option C
Ollama (Local, Free)
Run Ollama locally. Add OLLAMA_URL=http://localhost:11434. Pull a model: ollama pull llama3.2. Requires machine with 8GB RAM. 100% free, 100% private — no data leaves your machine.

🔨 로컬 개발 환경 설정 (Docker 없이)

개발 중 빠른 반복 작업을 위해 각 서비스를 개별적으로 실행합니다.

1. RAG 엔진 (Python)

cd rag-engine
python3.11 -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate
pip install -r requirements.txt
uvicorn main:app --port 8000 --reload

2. 백엔드 (.NET 8)

cd backend/DocuMind.Api
dotnet restore
dotnet run                        # http://localhost:5001 에서 시작

3. 프론트엔드 (React + Vite)

cd frontend
npm install
npm run dev                       # http://localhost:3001 에서 시작
시작 순서 서비스를 순서대로 시작합니다: RAG 엔진 먼저, 그 다음 백엔드, 그 다음 프론트엔드. RAG 엔진이 먼저 실행되지 않으면 .NET 백엔드가 상태 확인에 실패합니다.

🎯 엔드-투-엔드 워크플로

1
문서 업로드
/upload로 이동합니다. PDF, DOCX, TXT 또는 MD 파일을 드래그 앤 드롭하거나 클릭하여 업로드합니다. 파일당 최대 50MB. 한 번에 여러 파일을 일괄 업로드할 수 있습니다.
2
처리 대기
DocuMind가 파이프라인을 자동으로 실행합니다: 텍스트 추출 → 재귀적 청킹 → 임베딩 → ChromaDB 벡터 저장. 문서 상태가 실시간으로 업데이트되는 것을 확인합니다.
3
질문하기
/chat으로 이동합니다. 자연어로 질문을 입력합니다. DocuMind가 쿼리를 임베딩하고, ChromaDB에서 의미적으로 유사한 청크를 검색하여 출처 인용과 함께 답변을 반환합니다.
4
출처 검토
각 답변에는 인용 카드가 포함됩니다. 인용을 클릭하면 답변 생성에 사용된 정확한 문서 청크와 유사도 점수를 확인할 수 있습니다.
5
지식 베이스 탐색
/knowledge-base로 이동합니다. 인덱싱된 문서에 대해 직접 의미 검색을 실행합니다. 문서별로 필터링하고, 모든 청크를 확인합니다.
6
분석 보기
/analytics로 이동합니다. Recharts 시각화를 통해 사용 패턴, 쿼리 빈도, 가장 많이 접근된 문서, 시스템 성능 지표를 확인합니다.

🧪 RAG 파이프라인 직접 테스트 (curl)

프론트엔드와 백엔드와 독립적으로 RAG 엔진 API를 테스트하는 명령들입니다.

상태 확인

curl http://localhost:8000/health

문서 처리

curl -X POST http://localhost:8000/process \
  -H "Content-Type: application/json" \
  -d '{
    "file_path": "./uploads/test.txt",
    "document_id": "doc-001",
    "file_name": "test.txt"
  }'

지식 베이스 쿼리

curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{
    "query": "주요 주제가 무엇인가요?",
    "top_k": 5,
    "score_threshold": 0.3
  }'

컬렉션 목록 조회

curl http://localhost:8000/collections

문서 청크 조회

curl http://localhost:8000/chunks/doc-001

📋 사용 사례

UC-01
기업 지식 베이스
회사 정책, HR 문서, SOP, 내부 가이드를 업로드합니다. 직원들이 자연어로 질문할 수 있습니다: "재택근무 정책이 무엇인가요?" 또는 "휴가를 어떻게 신청하나요?" — 전체 문서를 읽지 않고도 즉시 인용된 답변을 얻을 수 있습니다.
UC-02
법률 문서 분석
계약서, NDA, 컴플라이언스 문서를 업로드합니다. 정확한 법률 용어를 알지 못해도 의미적으로 조항을 검색합니다 — "책임에 관한 모든 조항 찾기". Due diligence 및 법률 검토 워크플로에 적합합니다.
UC-03
기술 문서 Q&A
API 문서, 런북, 아키텍처 가이드, 인시던트 플레이북을 업로드합니다. 엔지니어들이 운영 질문을 합니다: "결제 서비스를 어떻게 재시작하나요?" 또는 "P1 인시던트 에스컬레이션 절차는?" — 정확한 출처 참조와 함께 답변됩니다.
UC-04
연구 보조
학술 논문, 연구 보고서, 시장 분석을 업로드합니다. 고수준 질문을 합니다: "트랜스포머 아키텍처에 관한 주요 발견은?" — 심층 독서를 위한 특정 섹션을 가리키는 출처 인용 요약을 얻습니다.
UC-05
파라미터 조정 및 실험
서비스를 재시작하지 않고 /settings 페이지를 사용하여 chunk_size, top_k, score_threshold를 조정합니다. 다양한 설정에 걸쳐 검색 품질을 비교합니다. 특정 문서 코퍼스에 대해 RAG 성능을 최적화하는 데 이상적입니다.

🧠 LLM 통합 옵션

LLM은 선택 사항입니다 DocuMind는 규칙 기반 답변 조합을 사용하여 LLM 없이 완전히 작동합니다. 자연어 생성(더 유창한 응답)이 필요한 경우에만 LLM을 추가하십시오. RAG 검색 품질은 LLM 선택과 무관합니다.
옵션 A
OpenAI
RAG 엔진 .env에 OPENAI_API_KEY=sk-...를 추가합니다. 권장 모델: gpt-4o-mini (가장 저렴, ~$0.15/1M 토큰). 최고 품질에는 gpt-4o. 무료 체험 크레딧 이용 가능.
옵션 B
Anthropic Claude
RAG 엔진 .env에 ANTHROPIC_API_KEY=sk-ant-...를 추가합니다. 권장 모델: claude-haiku-4-5 (가장 빠르고 저렴). 최고 품질에는 claude-sonnet-4-6.
옵션 C
Ollama (로컬, 무료)
Ollama를 로컬에서 실행합니다. OLLAMA_URL=http://localhost:11434를 추가합니다. 모델 다운로드: ollama pull llama3.2. 8GB RAM 필요. 100% 무료, 100% 프라이빗.

🔧 개발 환경 요구 사항

핵심 기능에 외부 API 키가 필요하지 않습니다 임베딩 모델(all-MiniLM-L6-v2)은 로컬 CPU에서 실행됩니다. OpenAI, Anthropic 또는 유료 서비스가 필요하지 않습니다. 인터넷은 최초 실행 시 모델 다운로드(~80MB)에만 한 번 필요합니다.
