🎯 Muammo bayoni

Nima uchun an'anaviy hujjat qidiruvi AI savol-javob uchun mos kelmaydi

Tashkilotlar ulkan hujjat kutubxonalarini to'playdi — PDF, Word fayllar, texnik qo'llanmalar, tadqiqot maqolalari — ammo bu hujjatlardan aniq javoblar olish qiyin bo'lib qolmoqda. Kalit so'z qidiruvi kontekstsiz ko'plab natijalar qaytaradi; LLM-lar asossiz "gallyutsinatsiya" qiladi; va ko'pchilik RAG yechimlar har bir so'rov uchun qimmat bulut API chaqiruvlarini talab qiladi.

  • Kalit so'z qidiruvi javob emas, hujjat sarlavhalarini qaytaradi — foydalanuvchilar tegishli ma'lumotni topish uchun butun hujjatlarni o'qishlari kerak.
  • Qidiruvsiz LLM-lar o'qitish ma'lumotlarida bo'lmagan faktlarni "gallyutsinatsiya" qiladi va ishonchli noto'g'ri javoblar beradi.
  • Bulut asosidagi RAG yechimlari uchinchi tomon API-lariga sezgir hujjat mazmunini yuboradi va ma'lumotlar maxfiyligini xavf ostiga qo'yadi.
  • Iqtibos mexanizmi yo'q: hatto AI to'g'ri javob bergan taqdirda ham foydalanuvchilar qaysi manba hujjatidan ekanligini tekshira olmaydi.
  • Katta miqyosda yuqori so'rov narxi: bulut embedding API-lari katta hujjat to'plamlari uchun oyiga minglab dollar turishi mumkin.

✨ Asosiy xususiyatlar

Tashqi API bog'liqligsiz intellektual hujjat savol-javob

📂
Ko'p formatli hujjat yuklash
PDF, DOCX, TXT va Markdown-ni qo'llab-quvvatlaydi. Matn ajratib olish formatga mos: PDF uchun PyMuPDF, Word hujjatlari uchun python-docx.
✂
Rekursiv bo'laklash
Hujjatlar 50 belgilik ustma-ust bilan 512 belgilik bo'laklarga bo'linadi. Ustma-ust bo'lak chegaralarida kontekst yo'qolishini oldini oladi.
🔢
Mahalliy Embeddinglar
Mahalliy ishlaydigan all-MiniLM-L6-v2 (384 o'lchov) dan foydalanadi. Nol tashqi API chaqiruvlari — hujjatlaringiz infratuzilmangizdan chiqmaydi.
🔍
HNSW Vektor qidiruvi
ChromaDB ning HNSW indeksi LLM-siz 30–100ms kechikish bilan taxminiy eng yaqin qo'shni qidiruvini ta'minlaydi.
📎
Manba iqtiboslari
Har bir javob aniq manba hujjati va bo'lakka qaytaradigan iqtiboslarni o'z ichiga oladi. Foydalanuvchilar har bir da'voni asl matn bilan tekshirishi mumkin.
📊
Analytics Dashboard
So'rov hajmini, top hujjatlarni, o'rtacha dolzarblik ko'rsatkichlarini va vaqt o'tishi bilan so'rov kechikishini ko'rsatuvchi Recharts asosidagi analytics.
Tashqi API bog'liqliqlari yo'q: Embedding modeli sentence-transformers yordamida mahalliy ishlaydi. So'rov kechikishi LLM-siz taxminan 30–100ms ni tashkil etadi — LLM generatsiya qadamini qo'shish apparat holatiga qarab 1–5 soniya qo'shadi.

🚀 Tezkor boshlash

DocuMind-ni ishga tushirish — embedding modeli birinchi ishga tushirishda avtomatik yuklab olinadi

1-qadam: Klonlash va loyihaga kirish

git clone https://github.com/your-org/rag-documind.git cd rag-documind

2-qadam: Barcha xizmatlarni ishga tushirish (model birinchi ishga tushirishda ~90 MB yuklab olinadi)

docker compose up --build -d # Xizmatlar ishga tushmoqda: # ✓ frontend → http://localhost:3001 # ✓ dotnet-api → http://localhost:5001 # ✓ rag-engine → http://localhost:8000 # ✓ chromadb → http://localhost:8001 (ichki)

3-qadam: API orqali hujjat yuklash

curl -X POST http://localhost:5001/api/documents/upload \ -F "file=@my_document.pdf" \ -F "title=Mening hujjatim" # Qaytadi: {"id": "abc123", "chunks": 42, "status": "indexed"}

4-qadam: Savol berish

curl -X POST http://localhost:5001/api/query \ -H "Content-Type: application/json" \ -d '{"question": "Asosiy natijalar nima?", "top_k": 5}' # Qaytadi: {"answer": "...", "citations": [{"doc": "...", "chunk": "..."}]}

5-qadam: http://localhost:3001 da UI-ni oching

🏠 Mahalliy model bajarish

all-MiniLM-L6-v2 modeli to'liq on-premises ishlaydi. Hujjat mazmuni va so'rov matni infratuzilmangizdan chiqmaydi.

📁 Fayl turi tekshiruvi

Yuklangan fayllar ruxsat ro'yxatiga (PDF, DOCX, TXT, MD) qarshi tekshiriladi. Fayl mazmuni kengaytmadan tashqari tekshiriladi — MIME turi ham nazorat qilinadi.

📏 Fayl hajmi cheklovlari

Maksimal yuklash hajmi API gateway darajasida qo'llaniladi (sukut bo'yicha 50 MB). Katta fayllar parsing qatlamiga etishgunga qadar rad etiladi.

🗂 Vektor izolyatsiyasi

Har bir hujjatning vektorlari nomlangan ChromaDB to'plamida saqlanadi. Hujjatni o'chirish uning vektorlarini indeksdan butunlay olib tashlaydi.

🔗 Iqtibos yaxlitligi

Har bir javob foydalanuvchilarga AI tomonidan yaratilgan javoblarni asl matnlar bilan tekshirishga imkon beradigan bo'lak darajasidagi iqtiboslarni o'z ichiga oladi.

🐳 Tarmoq izolyatsiyasi

ChromaDB ichki Docker tarmog'ida ishlaydi va xostga ochiq emas. Faqat frontend va .NET API tashqaridan foydalanish mumkin.
